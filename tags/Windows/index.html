建立新目錄或一個新的空白檔案，可使用New-Item指令

複製目錄下所有資料

搬移檔案/目錄

使用Move-Item指令，-Destination參數使用方法同Copy-Item

刪除檔案/目錄

使用Remove-Item指令

(PowerShell)常見輸出訊息的方法

Write-Host

直接輸出到終端機畫面

Write-Output

將訊息傳遞出來，若無接收者，則輸出到終端機畫面

Write-Warning

輸出警告訊息，會搭配$WarningPreference來設定是否輸出訊息，$WarningPreference預設為Continue，因此會輸出訊息後繼續執行

Write-Debug

輸出偵錯訊息，會搭配$DebugPreference來設定是否輸出訊息，$DebugPreference預設為SilentlyContinue，因此不會輸出訊息，但會往下執行

Write-Error

除了輸出訊息到終端機外，還會將錯誤訊息加入到Error Stream(也就是$Error變數)，且不會終止當前動作，繼續往下執行

(PowerShell)壓縮/解壓縮

壓縮/解壓縮

指令 說明
Compress-Archive 壓縮
Expand-Archive 解壓縮

2個指令的常用參數相似，見以下說明。

壓縮整個資料夾

壓縮多個檔案

更新壓縮檔(Update)/強制覆蓋(Force)

指定壓縮層級(CompressionLevel)

level description
Fastest 最快速方式壓縮。(檔案較大)
NoCompression 不壓縮
Optimal 最佳化壓縮，此方式為預設值。(檔案較小)

若要減少壓縮時間，可以Fastest壓縮層級進行壓縮

指名檔案名稱(LiteralPath)

大部分狀況下使用-Path可順利進行壓縮，若遇檔案名稱含有特殊字元或萬用字元等狀況，無法順利壓縮時，可改用-LiteralPath參數

加密壓縮(7-Zip)

powershell內建的Compress-Archive指令並無加密功能，改用7-Zip可在壓縮時加上密碼。

Exit Codes from 7-Zip

Code Meaning
0 No error
1 Warning (Non fatal error(s)). For example, one or more files were locked by some other application, so they were not compressed.
2 Fatal error
7 Command line error
8 Not enough memory for operation
255 User stopped the process

(PowerShell)Exit Status

$?

包含最後一個命令的執行狀態。
如果最後一個命令成功，則會包含 True，如果失敗，則為 False。
Write-ErrorCmdlet 會在執行後將設定$?為false，但在呼叫它的函式中，不會將設定$?為false

$LASTEXITCODE

上次執行之原生程式的結束代碼。

若為原生命令 (可執行檔) ，當$LASTEXITCODE是0時，$?會設定為 True，當$LASTEXITCODE為任何其他值，則設定為 False。
